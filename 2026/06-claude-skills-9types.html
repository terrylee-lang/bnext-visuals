Claude Code Skill 的 9 種類型

Anthropic 盤點內部數百個活躍 skill 後，歸納出這 9 種類型。它不是標準答案，而是一張檢查地圖，幫你看出自己的團隊還缺哪一種。

資料來源：Anthropic《Lessons from building Claude Code》 | 製作：《數位時代》
01
函式庫與 API 參考

教 Claude 正確使用某個函式庫、CLI 或 SDK，並附上常踩的雷。

02
產品驗證

描述怎麼測試、驗證程式碼有沒有真的動。Anthropic 說這類對產出品質可量測的影響最大，值得花一週做到極好。

03
資料抓取與分析

接上你的資料與監控系統，內含抓資料的腳本與儀表板代號。

04
業務流程與團隊自動化

把每日回報、建立任務等重複工作流，收成一個指令。

05
程式碼鷹架與範本

自動生出新服務、新模組的樣板，不必每次從零打起。

06
程式碼品質與審查

強制團隊的程式風格，協助 code review。

07
CI/CD 與部署

顧 PR、跑測試、漸進部署，出問題自動回滾。

08
Runbooks（故障排除手冊）

接到一個症狀，走完多工具調查，產出結構化報告。

09
基礎設施操作

執行例行維運，對破壞性動作加上護欄。

最好的 skill 通常只落在一類；想一次做太多事的，會橫跨好幾類，反而把 Claude 搞混。對照這張地圖，先補你最缺的那一種。
