Loop Engineering · 迴圈工程拆解

五個階段繞成一圈，六塊組件各自餵養它

一個迴圈在概念上是五階段的循環，通過驗證就交付，沒通過再跑一次。你實際要動手「建」出來的，是六塊組件。點選任一組件，看它推動循環的哪一段。

概念來源：Addy Osmani、Boris Cherny、@0xCodez | 製作：《數位時代》

＋點選下方任一組件，循環中被它推動的階段會亮起

02Plan規劃
03Execute執行
04Verify驗證
05Iterate迭代

尚未選擇。每塊組件負責推動循環中不同的階段，點下方卡片看對應關係。 自動化是整個迴圈的心跳，照排程驅動五個階段全程，讓迴圈成為「迴圈」而非「跑過一次」。 Worktree給每個代理一個隔離的工作目錄，讓多個代理在執行階段平行跑而不互相覆蓋檔案。 技能（Skills）把專案知識寫一次、每輪都讀，是全階段共用的底層脈絡，知識會累積複利。 連接器（Connectors）建在 MCP 之上，讓迴圈在執行與驗證階段碰得到待辦系統、資料庫、測試 API 與 Slack。 子代理（Sub-agents）讓驗證誠實，檢查的人不是動手的人，抓出第一個代理「自己說服自己」的問題。 記憶（Memory）讓迴圈在迭代時跨次不失憶。代理會忘，倉庫不會忘，明天接著今天停下的地方繼續。
