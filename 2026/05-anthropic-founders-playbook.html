FIGURE 01 · 4 STAGES

每階段該用哪個 Claude？

① IDEA（概念）
目標 問題-解方適配
Chat Cowork
⚠ 典型失敗
把「建構」當「驗證」，42% 新創倒在打造沒人要的東西
★ 一個練習
請 Claude 當「魔鬼代言人」，要它找出你論點的弱點、為競爭對手寫一個會贏的論述
② MVP（最小可行產品）
目標 產品-市場適配
Code Code Security
⚠ 典型失敗
代理式技術債以複利速度累積，AI 每次會話都從頭推斷架構
★ 一個練習
寫一份 CLAUDE.md 架構脈絡文件，列出採用的模式、禁用的依賴、取捨。每次新會話先讓 Claude 讀
③ LAUNCH（發布）
目標 可重複的成長引擎
Cowork Code Security
⚠ 典型失敗
創辦人「事必躬親」變組織瓶頸：原本一小時的決定現在拖一週
★ 一個練習
用 Cowork 對營運層做盤點，每項任務分類成：「全自動化／給代理／真的需要創辦人判斷」
④ SCALE（規模化）
目標 累積成護城河
Projects Memory Skills
⚠ 典型失敗
以為程式碼是護城河——其實是累積的領域深度、整合與專屬資料
★ 一個練習
把腦內知識（產業術語、法規地雷、不可行解法）灌進 Claude Skills，建立通用 AI 追不上的專屬基底

註：手冊原文 4 階段為 Idea Stage、MVP Stage、Launch Stage、Scale Stage。Claude Code Security 目前為 limited beta（封閉測試版），使用前需向 Anthropic 確認資格。「42% 新創因打造沒人要的東西而失敗」為 Anthropic 引用 CB Insights 既有產業統計，非自家研究。
